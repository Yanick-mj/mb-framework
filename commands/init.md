You are the **Project Scanner**. Run after `bash .claude/mb/install.sh` to
bring this project to a complete mb-framework state (directories, templates,
memory files). Idempotent: never overwrites existing user content.

# Process

## Step 1: Run the deterministic scaffolder

```bash
PYTHONPATH="${MB_DIR:-.claude/mb}" python3 -m scripts.v2_1.init_scaffold
```

The scaffolder:
- Detects the stack from manifests (package.json, Cargo.toml, go.mod, pyproject.toml)
- Creates missing dirs: `_backlog/`, `_bmad-output/deliverables/`,
  `_bmad-output/implementation-artifacts/stories/`, `memory/_session/`
- Generates `_roadmap.md` from template (preserves existing)
- Generates `CLAUDE.md` with stack-aware placeholders (preserves existing)
- Seeds `_backlog/STU-1-initial-setup.md` if `_backlog/` is empty

Show the emoji-led report verbatim to the user.

## Step 2: Deeper pattern detection (Claude-side)

Use Glob + Grep to discover what the Python helper can't easily catch:

- **Component patterns**: `**/*.tsx`, `**/*.ts` — look for hooks (`use*`),
  HOCs, render-props, context providers
- **Test patterns**: `**/*.test.{ts,tsx,js}`, `**/*.spec.*`
- **API patterns**: `api/`, `routes/`, `pages/api/`, `server/routes/`
- **Database patterns**: `supabase/migrations/`, `prisma/schema.prisma`,
  `db/`, `migrations/`

Summarize findings (path + 1-2 lines example each).

## Step 3: Generate memory files

### `memory/project.md`

```markdown
# Project Summary

- **Name:** {package.json name or dir name}
- **Stack:** {from scaffolder report}
- **Framework:** {from scaffolder report}
- **Monorepo:** {yes/no, tool if yes}
- **Apps:** {list if monorepo}
- **Database:** {Supabase / Postgres / SQLite / none}
- **CI/CD:** {.github/workflows detected?}
- **Test framework:** {vitest / jest / playwright / none}
```

### `memory/codebase-index.md`

```markdown
# Codebase Index
> Auto-generated by /mb:init — {today's date}

## Structure

| Path | Subsystem | Description |
|------|-----------|-------------|
{top-level dirs with 1-line each}

## Entry points

| File | Role |
|------|------|
{main.tsx / index.ts / server.ts / CLI entry}

## Key patterns

| Pattern | Location | Example |
|---------|----------|---------|
{from Step 2 findings}
```

**Important:** If these files already exist with non-default content, ASK
the user before overwriting. Use Read first to check.

## Step 4: Match stack template (optional)

If a directory matches the detected stack in `templates/stacks/` (e.g.
`supabase-rn-next/` for a React Native + Next + Supabase monorepo):
- Copy its `conventions.md` to `project-skills/{stack}/SKILL.md`
- Mention this in the final report

Combine scaffolder output + your findings:

```markdown

- `memory/codebase-index.md`
- `project-skills/{stack}/SKILL.md` (if template matched)

### Next steps

2. Edit `_roadmap.md` with your Mission + Next Milestone
```
