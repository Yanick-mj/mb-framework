# Skills

Skills are injectable domain knowledge files. They're NOT agents (no persona, no decision-making). They're reference material that agents read when needed.

## How Skills Work

1. The orchestrator detects that a task touches a specific domain
2. It tells the agent: "Read skill X before starting"
3. The agent uses the skill's knowledge to inform its work

## Creating a Skill

Create a directory with a SKILL.md inside:

```
skills/
└── my-domain/
    └── SKILL.md
```

The SKILL.md needs frontmatter:

```yaml
---
name: 'mb-skill-{name}'
description: '{what this skill provides}'
when_to_use: '{when an agent should read this}'
---
```

Then write the domain knowledge as structured markdown.

## Project-Specific Skills

Put project-specific skills in `project-skills/` (not here).
These are generated by `/mb:init` or created manually.
