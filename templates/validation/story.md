# Story Validation Checklist

## Structure
- [ ] Title follows pattern: "As a {role}, I want {action}, so that {value}"
- [ ] Acceptance criteria are specific and testable
- [ ] Each AC has a clear pass/fail condition

## Scope
- [ ] Single responsibility (one feature per story)
- [ ] Estimatable (not too large)
- [ ] Independent (no hidden dependencies)
- [ ] Valuable (delivers user value)

## Technical Readiness
- [ ] Required data model documented
- [ ] API contracts defined
- [ ] File paths identified
- [ ] Migration needed? (yes/no)
- [ ] RLS impact assessed

## Testing
- [ ] Test approach defined
- [ ] Edge cases listed
- [ ] Performance criteria if applicable
